## üè∑Ô∏è KODING SISTEM

### Struktur Kode SPPG:
```
Format: SPPG-[Kategori][Subkategori][Nomor]
Contoh: SPPG-B01-001 (Beras - Premium 1kg)
```

### Kategori Utama:
- **B** = Bahan Pokok (Staple Foods)
- **K** = Produk Kemasan (Packaged Products)
- **M** = Bumbu & Rempah (Spices & Seasonings)
- **O** = Produk Olahan (Processed Foods)
- **D** = Daging & Seafood (Meat & Seafood)
- **T** = Bahan Tambahan (Additional Materials)

### Model Data Mapping di Database

```sql
CREATE TABLE sppg_material_plu_mappings (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    material_code VARCHAR(50) NOT NULL,
    plu_number VARCHAR(10) NOT NULL,
    mapping_type ENUM('direct', 'approximation') NOT NULL DEFAULT 'direct',
    conversion_factor DECIMAL(10,4) NOT NULL DEFAULT 1.0000,
    is_primary BOOLEAN NOT NULL DEFAULT TRUE,
    notes VARCHAR(255) NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    CONSTRAINT fk_sppg_mapping_material FOREIGN KEY (material_code) REFERENCES sppg_materials(material_code),
    CONSTRAINT fk_sppg_mapping_plu FOREIGN KEY (plu_number) REFERENCES plu_codes(plu_number),
    INDEX idx_sppg_mapping_material (material_code),
    INDEX idx_sppg_mapping_plu (plu_number),
    INDEX idx_sppg_mapping_primary (is_primary)
);
```

### View Gabungan Material, PLU, dan Nutrisi

```sql
CREATE VIEW v_sppg_material_plu_nutrition AS
SELECT
    sm.material_code,
    sm.material_name,
    sm.category,
    sm.subcategory,
    sm.unit,
    sm.package_size,
    mpm.plu_number,
    pc.product_name AS plu_product_name,
    pc.category AS plu_category,
    pc.subcategory AS plu_subcategory,
    sm.protein_per_100g,
    sm.carb_per_100g,
    sm.fat_per_100g,
    sm.fiber_per_100g,
    sm.calories_per_100g
FROM sppg_materials sm
LEFT JOIN sppg_material_plu_mappings mpm
    ON mpm.material_code = sm.material_code
    AND mpm.is_primary = TRUE
LEFT JOIN plu_codes pc
    ON pc.plu_number = mpm.plu_number
WHERE sm.is_active = TRUE;
```

```sql
    shelf_life_months INT NOT NULL,
    supplier_id BIGINT UNSIGNED NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## ÔøΩ KALKULATOR KEBUTUHAN GIZI

### Formula Perhitungan:
```sql
CREATE VIEW v_sppg_nutrition_calculator AS
SELECT 
    material_code,
    material_name,
    category,
    subcategory,
    package_size,
    protein_per_100g,
    carb_per_100g,
    fat_per_100g,
    fiber_per_100g,
    calories_per_100g,
    -- Hitung kebutuhan per orang per hari
    CASE 
        WHEN target_group = 'anak' THEN 
            (protein_per_100g * 30 / 100) * 7 + -- 30g protein/hari
            (carb_per_100g * 130 / 100) * 7 + -- 130g carb/hari  
            (fat_per_100g * 25 / 100) * 7 + -- 25g lemak/hari
            (fiber_per_100g * 25 / 100) * 7 -- 25g serat/hari
        WHEN target_group = 'dewasa' THEN
            (protein_per_100g * 50 / 100) * 7 + -- 50g protein/hari
            (carb_per_100g * 300 / 100) * 7 + -- 300g carb/hari
            (fat_per_100g * 65 / 100) * 7 + -- 65g lemak/hari
            (fiber_per_100g * 30 / 100) * 7 -- 30g serat/hari
    END as daily_protein_grams,
    calories_per_100g * 2000 / 100 * 7 as daily_calories
FROM sppg_materials
WHERE is_active = TRUE;
```
